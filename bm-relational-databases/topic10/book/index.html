Objectives

Practice SQL SELECT statement joining to multiple tables.

ON and USING clause

Login to applicaiton express

https://iacademy3.oracle.com/

Choose SQL Workshop and then SQL Commands

Write the necessary SQL SELECT statements for the following queries. Tor each query consider the output, practice your use of column and table aliases, concatenation and ordering of output. You will also need to investigate the tables to find the correct column names and foreign keys etc. Use the Object Browser to do this.

  • Join the locations and departments table using the location_id column.
  • Join DJs on Demand d_play_list_items, d_track_listings and d_cds tables with the JOIN USING syntax. Include the song ID, CD number, title, and comments in the output.
  • Display the city, department name, location ID, and department ID for departments 10, 20, and 30 for the city of Seattle.
  • Display country name, region ID and region name for Americans.
  • Write a statement joining the employees and jobs tables. Display the first and last names, hire date, job id, job title and maximum salary. Limit the query to those employees who are in jobs that can earn more than 12000.
  • Display job title, employee first name, last name, and email for all employees who are stock clerks.
  • Use JOIN ON syntax to query and display the location ID, city, and department name for all canadian locations.
  • Query and display manager ID, department ID, department name, first name, and last name for all employees in department 80, 90, 100 and 190).
  • Display employee ID, last name, deparment ID, department name and hire date for those employees whose hire date was June 7, 1994.

OUTER JOINS

Practice OUTER joins

  • Return the first name, last name and department name for all employees including those employees not assigned to a department.
  • Return the first name, last name and department name for all employees including those departments that do not have an employee assigned to them.
  • Return the first name, last name and department name for all employees including those departments that do not have an employee assigned to them and those employees not assigned to a department.
  • Query the DJs on Demand database to return the first name, last name, event date, and description of the event the client held. Include all the clients even if they have not had an event scheduled.
  • Using the Gloabl Fast Foods database, show the shift description and shift assignment date even if there is no date assigned for each shift description.

SELF JOINS

Practice SELF JOINS

  • Display the employee's last name and employee number along with the manager's last name and manager number. Label the columns Employee, Emp#, Manager, and Mgr#
  • Modify the first query to display all employees and their manager, even if the employee does not have a manager. Order the list alphabetically by the last name of the employee.
  • Display the names and hire dates for all employees who were hired before their managers, along with their managers' names and hire dates. Label the columns Employee, Emp hired, Manager, and Mgr Hired.
