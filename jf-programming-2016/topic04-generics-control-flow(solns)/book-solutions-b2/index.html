Exercise 8 (Advanced)

Consider this list of primitive type double values:

    double d1 = 100.01;
    double d2 = 311.64;
    double d3 = 206.36;
    double d4 = 721.64;
    double d5 = 951.56;
  • Store these values in an ArrayList.
    • Warning: primitives cannot be stored directly in an ArrayList.
      • ArrayList : incorrect
      • ArrayList : correct

Use a method with signature public void convertToArrayList()

Print out the list of elements using an iterator to traverse the ArrayList.

Solution

  • Open the algorithm project contained in your folder workspaceBlueJ/session04.
  • Open the Collections class source code in the editor.
    • Add a method public void convertToArrayList in stages below, compiling each stage.

    {

    }
  • Add the sample data :

        double d2 = 311.64;
        double d3 = 206.36;
        double d4 = 721.64;

Declare and allocate memory to an ArrayList.

    public void convertToArrayList()
    {   
        double d1 = 100.01;
        double d2 = 311.64;
        double d3 = 206.36;
        double d4 = 721.64;
        double d5 = 951.56;

        ArrayList<Double> listDouble = new ArrayList<>();

An import statement is required for the ArrayList:

import java.util.ArrayList;

We can add a double type directly to the ArrayList array with the following statement:

listDouble.add(d1);

where listDouble is the ArrayList and d1 is a primitive double type variable as above.

This is possible thanks to a Java feature known as Autoboxing. See here for detailed information in the Java Tutorials series on this topic.

The opposite of Autoboxing is Unboxing. Here is an example:

double dval = listDouble.get(4);

In this example, again, listDouble is the ArrayList and so its elements are type Double : yet, as can be seen, one of its Double elements can legally be assigned to a primitive double variable.

    public void convertToArrayList()
    {   
        double d1 = 100.01;
        double d2 = 311.64;
        double d3 = 206.36;
        double d4 = 721.64;
        double d5 = 951.56;

        ArrayList<Double> listDouble = new ArrayList<>();

        listDouble.add(d1);
        listDouble.add(d2);
        listDouble.add(d3);
        listDouble.add(d4);
        listDouble.add(d5);
    }

Finally, we print out the values using an iterator as required.

Here is the complete method.

    public void convertToArrayList()
    {   
        double d1 = 100.01;
        double d2 = 311.64;
        double d3 = 206.36;
        double d4 = 721.64;
        double d5 = 951.56;

        ArrayList<Double> listDouble = new ArrayList<>();

        listDouble.add(d1);
        listDouble.add(d2);
        listDouble.add(d3);
        listDouble.add(d4);
        listDouble.add(d5);

        Iterator<Double> it = listDouble.iterator();
        while(it.hasNext())
        {
            System.out.println("Value of double variable is "+it.next());
        }
    }

Exercise 9

Write a class OrderCheck4 using the class OrderCheck3 that was developed in the lab as a model.

The purpose of the class is to:

  • Accept 4 integers as constructor parameters
  • Provide a method to check if the parameters are ordered
    • The signature of the method is
      • public boolean isOrdered()

Solution

Here is the completed OrderCheck3 class developed in the lab:

public class OrderCheck3
{
    int a;
    int b;
    int c;
    /**
     * Constructor
     * @param a an integer
     * @param b an integer
     * @param c an integer
     */
    public OrderCheck3(int a, int b, int c)
    {
        this.a = a;
        this.b = b;
        this.c = c;
    }

   public boolean isOrdered()
   {
       return (a < b && b < c) || (a > b && b > c);
   }

}

In the algorithm project create a new class OrderCheck4.

Introduce 4 integer fields and a constructor to initialize these:

{
    int a;
    int b;
    int c;
    int d;

    public OrderCheck4(int a, int b, int c, int d)

        this.b = b;

Study the method isOrdered in OrderCheck3 and apply the same logic but for 4 rather than 3 variables.

Here is the method we require:

   public boolean isOrdered()
   {
       return (a < b && b < c && c < d) || (a > b && b > c && c > d);
   }

We check both ascending and descending order.

Notice that we have surrounded the return statement parts with parentheses.

The reason for doing so is to comply with a recommendation in the style guide. See Figure 1, an extract from Java Code Conventions.

The reason that parentheses are not functionally required is to be found by studying the table of Operator Precedence here, also shown below in Figure 2.

Notice that the relational operators < and > have higher precedence than the logical operators && and || and so, for example, a < b and b < c are evaluated before the && operation is applied which is what we require.

Here is the complete class

    int b;
    int c;
    int d;

    public OrderCheck4(int a, int b, int c, int d)
    {
        this.a = a;
        this.b = b;
        this.c = c;
        this.d = d;
    }

    public boolean isOrdered()
    {
       return (a < b && b < c && c < d) || (a > b && b > c && c > d);
    }
}

Solution

Let's consider the missing code in the first else if block:

        else if (...) // then b is smallest
        {
            if(...) // sorted order is b a c
            {
                ...             
            }
            else// sorted order is b c a
            {
                ...              
            }
        }

If b is the smallest

  • then it is less than both a and c
  • therefore b < a && b < c

This allows us to fill in the missing code in the else if(...) statement:

else if (b < a && b < c)

Notice that it is not necessary to surround b < a nor b < c with parentheses because, as previously discussed, the precedence of relational operators is greater than logical operators and so b < a will be evaluated first rather than, for example a && b being evaluated first which, were this to be the case, would lead to an incorrect result.

Next consider the missing code from:

            if(...) // sorted order is b a c
            {
                ...             
            }

Here we are considering the case where the order is b a c :

  • so the test is a < c.
  • if a is less than c
    • then print the order b a c
            if(a < c) // b a c
            {
                System.out.println("" + b + " " + a + " " + c);                
            }

The else block follows logically:

  • We know that here b is the smallest
  • The two possible orderings are therefore
    • b a c
    • b c a
  • Therefore the order in this, the else block must be b c a since we have already encountered b a c.

Here then is the completed else if block:

        else if (b < a && b < c) // then b is smallest
        {
            if(a < c) // b a c
            {
                System.out.println("" + b + " " + a + " " + c);                
            }
            else// b c a
            {
                System.out.println("" + b + " " + c + " " + a);                
            }
        }

Applying similar reasoning to the the last else if block produces:

        else if (c < a && c < b) // then c is smallest
        {
            if(a < b) // c a b
            {
                System.out.println("" + c + " " + a + " " + b);                
            }
            else// c b a 
            {
                System.out.println("" + c + " " + b + " " + a);                
            }
        }

Exercise 13

Study both methods in ThreeSort: sort and sort2.

  • Which method
    • is easier to understand and explain to others?
    • would you prefer to maintain?

Solution

Here is the method sort:

    public static void sort(int a, int b, int c)
    {
        // determine smallest and largest of three
        int smallest = Math.min(a, Math.min(b, c));
        int biggest = Math.max(a, Math.max(b, c));

        //discover the middle number   
        int tmp1 = Math.min(Math.max(a, b), Math.max(a, c));
        int tmp2 = Math.min(Math.max(a, b), Math.max(b, c));
        int middle = Math.min(tmp1, tmp2); 

        System.out.println("" + smallest + " " + middle + " " + biggest);  
    }

Here is the method sort2:

public static void sort2(int a, int b, int c)
    {
        if(a < b && a < c) //then a is smallest
        {
            if(b < c)// a b c
            {

            }
            else //a c b
            {
                System.out.println("" + a + " " + c + " " + b);
            }
        }
        else if (b < a && b < c) // then b is smallest
        {
            if(a < c) // b a c
            {
                System.out.println("" + b + " " + a + " " + c);                
            }
            else// b c a
            {
                System.out.println("" + b + " " + c + " " + a);                
            }
        }  
        else if (c < a && c < b) // then c is smallest
        {
            if(a < b) // c a b
            {
                System.out.println("" + c + " " + a + " " + b);                
            }
            else// c b a 
            {
                System.out.println("" + c + " " + b + " " + a);                
            }
        }

Certainly the first method is the more elegant but whether it is easier to understand and thus easier to maintain is a question that is left to you to answer.
