Welcome

Welcome to the Waterford Institute of Technology (WIT) ICTSkills Summer School, also referred to as ICTSkills Studio. The course spans six consecutive weeks commencing late May and ending early July. The venue is the WIT West Campus at Carriganore. Course content includes the git versioning system, an introduction to the JavaScript language, dynamic web development and a module on oral and written communications. The course is a five credit module. Grading is based on continuous assessment and a web application development project.
Venue

The course will be run on Tuesdays and Wednesdays and industry presentations will take place on Thursdays in the boardroom at Carriganore. You may view maps of the location and route in the next step. Accommodation will also be available elsewhere for work on Mondays and Fridays and for the period between the end of the course and beginning of the autumn semester at a venue or venues to be advised.
Outline

Here we present an outline timeline and the assignment grading rubric.
Basic-Git

Lab describing how to use git to manage a project by creating a local repository, inserting the project files into the repository and creating and managing a tracking copy of the repository on a cloud-based server.
Donation

Develop a dynamic web application called donation that will allow a user to register, log in and log out, make donations and print a report of all donations.

JavaScript-I

Introducing the fundamentals of the javascript language, becoming familiar with some of the different sources of a selection of javascript resources. Explore Google Chrome's javascript-based tools. Understand basic types. Acquire an understanding of the roles that arrays and objects play in a javascript program. All using JetBrains' WebStorm IDE.
JavaScript-II

Examine some of JavaScript's constructs such as conditionals, loops, arrays, objects, functions and scope. Understanding some concepts such as Truthy and Falsy values. simple static web site, include an external JavaScript file and use Google Chrome developer tools to debug into the script at runtime.
MyRent

myrent, a prototype web application to allow a user to register and authenticate, to gather data on residential properties, to store the data in a database and generate reports on properties located within specified regions.
JavaScript III

Use both native JavaScript and JQuery to access and manipulate the DOM, for example by hiding and revealing nodes (elements).
JavaScript IV (Maps)

Demonstrate how to embed a map in a web page and add a marker or overlay. Enable the marker so that it may be dragged and dropped. Capture the marker's geolocation and dynamically display this on the webpage.

Circle-map-overlay

Use JavaScript to manipulate map using resizable and moveable circular overlays.
Polygon-map-overlay

Manually create a polygon-shaped overlay on a Google map and filter out marker overlays located outside polygon. Employ a jQuery ajax call to fetch users and their geolocations from the server.
JavaScript VI (SemanticUI)

Semantic elements such as dropdown and radio button require initialization. Semantic also facilitates form validation. In this short tutorial a dropdown is initialized, form validation is invoked and use of a Semantic progress bar is demonstrated. jQuery ajax is employed to fetch and render data. Warning: This lab applies to Semantic UI version 1x. See the appendix for Semantic 2 material.
Ajax

Asynchronous JavaScript & XML. Used to manage transmission of data between client (web) and server. The ajax operation takes place in the background. Facilitates updates of targetted areas on page without requirement to refresh entire page.
Advanced-Git-I

The lab describes how a two-person team might collaborate to use Git to manage a project. Individually owned and shared material is used to demonstrate where conflicts may be expected and how to resolve them when they arise.
Advanced-Git-II

We explore how merging and rebasing may be used to manage Git history. Use of simple examples should provide confidence in using these features to produce a professional-looking project history.
TDD

A Play model set composed of three entity-related classes is developed using a Test Driven Development (TDD) approach. The individual model classes are developed within individual test classes. The tests may be run individually or as a group within a test suite, the development of which is the final step in the lab.
WitPress

witpress, a prototype web application that allows an authenticated user to manage blog posts and comments on posts. An authenticated user may create a blog post, delete from its existing list of posts, add comments and access the public blog directory.

Project (Resources)

Assignment: The submission process comprises a cover page to be uploaded by means of a dropbox on Moodle. On the page should be links to the three project repositories on Bitbucket and the url to the completed application, deployed on Heroku. A link to download a cover page template is provided in this resource.
Assignment Tips

This short lab provides some hints on how to implement the more difficult parts of the assignment. The topics discussed are Ajax, Global abatement (maps), validation and manipulation of Semantic controls (dropdown and edit box), dynamic updating of map markers, specifying alternative Play templates and, lastly, deployment of a non-master branch to Heroku.
Heroku

Tutorial on deployment of a Play web application to the cloud (Heroku).
