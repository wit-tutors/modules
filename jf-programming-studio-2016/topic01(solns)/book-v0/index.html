Create baseline project skeleton using Play 1.3.0

Use the Play Framework to create the application

  • A suggested folder layout for the ICTSkills Studio program is illustrated in Figure 1.
    • This provides for the thee applications, the subject of the ICTSkills Studio program.
      • Donation
      • MyRent
      • WitPress

  • Create a new Play application named donation in your workspace as follows:
    • It is assumed that you already have Play installed and configured.
    • Open a terminal window, change directory to your workspaceStudio.
    play new donation

Accept the default prompts.

  • Once this has completed, change into the folder just created (donation).

  • Then run the eclipsify command:

    play eclipsify

This generates the necessary infrastructure to allow the application be imported as a project in Eclipse.

  • Import the project into eclipse:

  • Note: Select the Java perspective in Eclipse - not JavaEE.

  • Launch Eclipse and if necessary switch workspace to workspaceStudio.

  • In Eclipse, import the newly created application, Donation.

      File | Import | General | Existing Projects into Workspace.

  • Browse to Donation.

    • Since Donation is already in the workspace, untick "Copy projects into workspace" and click Finish.

    • It may be necessary to refresh using F5 to enable Next or Finish buttons.

  • Test the application by executing play run from terminal within the donation directory.

  • Switch to a browser, enter localhost:9000 in the location bar and press return. The Play welcome screen should then load.

In a terminal create a git repository and initialize as follows.

  • Note that you will first require to locate a .gitignore file in your donation root directory as described in the Basic-Git lab.
git init 
git add .
git status

Then commit the baseline project:

git commit -m 'baseline app created with Play 1.4.2'

Semantic UI & jQuery

Download and expand the file Semantic-UI-CSS-master.zip from here to a temporary directory.

  • The file expands to a folder named Semantic-UI-CSS-master.
    • Within this folder is an archive dist.
    • Copy dist to a temporary folder, expand and rename as semantic.
    • Copy semantic into public folder in the donation app Package Explorer.

Download the file jquery-2.2.3.zip to a temporary location and inflate.

In Eclipse, navigate to the public/javascripts folder and

  • Delete the auto-generated jquery (an earlier version).
  • Copy in jquery-2.2.3.js

Open views/main.html, if necessary changing the character encoding to UTF-8.

Add this link to the semantic css file directly under the meta element:

<link rel="stylesheet" media="screen" href="@{'/public/semantic/semantic.min.css'}">

Modify the jquery script tag to this:

    <script src="@{'/public/javascripts/jquery-2.2.3.js'}" charset="${_response_encoding}"></script>

main.html now becomes:

<!DOCTYPE html>
<html>

  <head>
    <title>#{get 'title' /}</title>
    <meta charset="utf-8">
    <link rel="stylesheet" media="screen" href="@{'/public/semantic/semantic.min.css'}">
    <script src="@{'/public/javascripts/jquery-2.2.3.js'}" charset="${_response_encoding}"></script>
    <script src="@{'/public/semantic/semantic.js'}"></script>
  </head>

  <body>
      #{doLayout /}
  </body>

</html>

In the view file Application.index replace #{welcome /} with Hello Donation

Connect to a local database in memory, provided as part of the Play framework. To do this, open the file application.conf which is located in the conf folder.

Locate the following code:

# To quickly set up a development database, use either:
#   - mem : for a transient in memory database (H2 in memory)
#   - fs  : for a simple file written database (H2 file stored)
# db.default=mem

Remove the # that preceeds db.default=mem.

  • Save the file.
  • In a terminal from within the donation folder execute play run.
  • Launch app in a browser
    • localhost:9000

Hello Donation should be displayed.

Iteration v0: Create a Model

User

The User Model class persists the user details such as its name, email and password.

It contains 2 methods: one to check if a supplied password matches the User's, the other to find a User instance given an email.


  package models;

  import java.util.ArrayList;
  import javax.persistence.CascadeType;
  import javax.persistence.Entity;
  import java.util.ArrayList;
  import java.util.List;
  import play.db.jpa.Model;

  @Entity
  public class User extends Model
  {
    public boolean usaCitizen;
    public String firstName;
    public String lastName;


    public String email;
    public String password;

    public User(boolean usaCitizen,
                  String firstName, 
                  String lastName, 
                  String email, 
                  String password
                  )
    {
        this.usaCitizen = usaCitizen;
      this.firstName = firstName;
      this.lastName = lastName;
      this.email = email;
      this.password = password;
    }

      public static User findByEmail(String email) 
    {
          return find("email", email).first();
      }

      public boolean checkPassword(String password) 
    {
          return this.password.equals(password);
      }
  }

The Accounts controller follows:


package controllers;

import play.*;
import play.mvc.*;
import java.util.*;
import models.*;

public class Accounts extends Controller 
{

    public static void index() 
    {
        render();
    }

    public static void signup() 
    {
        render();
    }

    public static void register(User user)
    {
        Logger.info(user.firstName + " " + user.lastName + " " + user.email + " " + user.password);
        user.save();
        index();
    }

    public static void login() 
    {
        render();
    }

    public static void logout() 
    {

        session.clear();
         index();
    }

    public static void authenticate(String email, String password) 
    {
        Logger.info("Attempting to authenticate with " + email + ":" + password);

        User user = User.findByEmail(email);
        if ((user != null) && (user.checkPassword(password) == true)) 
        {
            Logger.info("Successfull authentication of  " + user.firstName + " " + user.lastName);
            session.put("logged_in_userid", user.id);
        } 
        else 
        {
            Logger.info("Authentication failed");
            login();
        }
    }

    public static User getCurrentUser() 
    {
        String userId = session.get("logged_in_userid");
        if(userId == null)
        {
            return null;
        }
        User logged_in_user = User.findById(Long.parseLong(userId));
        Logger.info("In Accounts controller: Logged in user is " + logged_in_user.firstName);
        return logged_in_user;
    }
}
