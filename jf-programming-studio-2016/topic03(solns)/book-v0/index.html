Solution v0

Step-by-step guide to building MyRent application

  • Iteration v0 (Register)

Create baseline project skeleton using Play 1.3.0

Use the Play Framework to create the application

  • A suggested folder layout for the ICTSkills Studio program is illustrated in Figure 1.
    • This provides for the thee applications, the subject of the ICTSkills Studio program.
      • Donation
      • MyRent
      • WitPress

  • Create a new Play application named myrent in your workspace as follows:
    • It is assumed that you already have Play installed and configured.
    • Open a terminal window, change directory to your workspaceStudio.
    play new myrent

Accept the default prompts.

  • Once this has completed, change into the folder just created (myrent).

  • Then run the eclipsify command:

    play eclipsify

This generates the necessary infrastructure to allow the application be imported as a project in Eclipse.

  • Launch Eclipse and if necessary switch workspace to workspaceStudio.

  • In Eclipse, import the newly created application, MyRent.

      File | Import | General | Existing Projects into Workspace.
  • Import the project into eclipse:

  • Browse to MyRent.

    • Since MyRent is already in the workspace, untick "Copy projects into workspace" and click Finish.

    • It may be necessary to refresh using F5 to enable Next or Finish buttons.

  • Test the application by executing play run from terminal within the donation directory.

  • Switch to a browser, enter localhost:9000 in the location bar and press return. The Play welcome screen should then load.

Create a git repository, add and commit the project following steps similar to that described in detail in the solution to Donation v0 (step 01).

git init
git add .
git commit -m 'baseline created using Play 1.4.2'

Semantic UI

In the solution to Donation we used previously downloaded Semantic UI and jQuery modules. Here we shall use a content delivery network (CDN) for both.

Open views/main.html, if necessary changing the character encoding to UTF-8.

Replace the entire file with the following:

<!DOCTYPE html>

<html>
  <head>
    <title>#{get 'title' /}</title>
    <meta charset="${_response_encoding}">
    <link rel="stylesheet" type="text/css" href="http://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.1.8/semantic.min.css">
    <link rel="shortcut icon" type="image/png" href="@{'/public/images/favicon.png'}">
  </head>
  <body>
    #{doLayout /}       
    <script src="http://code.jquery.com/jquery-2.2.3.min.js"></script> 
    <script src="http://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.1.8/semantic.min.js"></script>          
    #{get 'moreScripts' /}
  </body>
</html>

In the view file Application.index replace #{welcome /} with "Hello MyRent".

Connect to a local database in memory, provided as part of the Play framework. To do this, open the file application.conf which is located in the conf directory.

Locate the following code:

# To quickly set up a development database, use either:
#   - mem : for a transient in memory database (H2 in memory)
#   - fs  : for a simple file written database (H2 file stored)
# db=mem

Remove the # that preceeds db=mem.

Save the file and run the application from within the myrent directory in a terminal window.

Then launch in a browser (localhost:9000).

You should be presented with the greeting Hello MyRent.

Iteration v0: Model (User)

Create a User model in the models package.

  • This model contains details of a user such as name, email and so on.
  • It facilitates data persistence, that is, saving the data to a specified database.

Here is the file content:

Filename: User.java

package models;

import javax.persistence.Entity;
import play.db.jpa.Model;

@Entity
public class User extends Model
{
  public String firstName;
  public String lastName;
  public String email;
  public String password;


  public static User findByEmail(String email)
  {
    return find("byEmail", email).first();
  }

  public boolean checkPassword(String password)
  {
    return this.password.equals(password);
  }

}

Iteration v0: Controllers (Accounts)

  • A controller class is required to associate the Accounts views with the User model and perform any necessary Actions.
    • An Action is a "function that handles a request and generates a result to be sent to the client" (Play Documentation).
  • The Accounts controller contains Actions to facilitate signing on (registering), logging in, authentication and so on.
  • Create the controller Accounts.java in the controllers package. Here is the file content:

Filename: Accounts.java


package controllers;

import models.User;
import play.Logger;
import play.mvc.Controller;

public class Accounts extends Controller
{

  public static void index()
  {
    render();
  }

  /**
   * Registers a user onto the MyRent system
   * 
   * @param firstName
   *          the user's first name
   * @param lastName
   *          the user's last name
   * @param email
   *          the user's email address
   * @param password
   *          the user's password, in plain text, naturally :-)
   */
  public static void register(User user)
  {
    user.save();
    login();
  }

  /**
   * Authenticates that a user's login credentials are valid
   * 
   * @param email
   *          The submitted email address
   * @param password
   *          The corresponding password
   */
  public static void authenticate(String email, String password)
  {
    User user = User.findByEmail(email);
    if ((user != null) && (user.checkPassword(password) == true))
    {
      Logger.info("Authentication successful");
    }
    else
    {
      Logger.info("Authentication failed");
      login();
    }
  }

  public static void login()
  {
    render();
  }

}
