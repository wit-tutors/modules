Play Framework

[1] What is a framework?

An abstraction in which common generic code may be overridden.

Distinguishing features:

  • inversion of control: In a framework, unlike in libraries or normal user applications, the overall program's flow of control is not dictated by the caller, but by the framework.

  • default behavior: A framework has a default behavior. This default behavior must actually be some useful behavior and not a series of no-ops.

  • extensibility: A framework can be extended by the user usually by selective overriding or specialized by user code providing specific functionality.

  • non-modifiable framework code: The framework code, in general, is not allowed to be modified. Users can extend the framework, but not modify its code.

Wikipedia: Software framework

[2] What is the Play Framework?

A framework to facilitate building web applications using Java or Scala.

[3] You have been developing using version 1.x. There is a later version 2.x. What are some fundamental differences? Would your version work without refactoring were you to upgrade to Play 2?

No. Version 2.x breaks backward compatibility, is a complete rewrite and uses a different language, Scala. Java is the language of 1.x

[4] Play uses the Model View Controller (MVC) pattern. Briefly explain the pattern. Why is it used? Is it unique to Play?

MVC is a widely used pattern. It separates presentation from the domain objects that model the real world. The model is unaware of the view. Presentation of the same model may be performed by different views, for example by a GUI and the command line. The presentation part of the MVC comprises the controller and view. The controller communicates with the view, receiving and manipulating user input data. Optionally it may transmit data to the view resulting from this manipulation. Clearly MVC is not unique to Play. Martin Fowler: Model View Controller.

[5] Each Play application contains a routes file. What is its purpose?

The routes file contains routing instructions for the built-in HTTP router. The router translates an incoming HTTP request to an action call. An action is a public controller method.

[6] Here is a typical extract from a routes file. Explain.

# Donation page
GET    /donation        DonationController.index
POST   /donation/donate DonationController.donate

In the above code block there are two HTTP requests.

For example:

  • /donation DonationController.index is the request path.
  • GET and POST are the HTTP methods. In the first example, the request results in the invocation of a method index in the controller class DonationController.

Play Documentation: HTTP Routing

[7] What is the purpose of the application.conf file?

The conf folder contains all configuration files for a Play app. One of these files is application.conf. This is the main configuration file containing standard configuration parameters such as the application name, identification and configuration of the database employed and so on.

Play Documentation: Configuration parameters

[8] The following Play command is frequently used. What is the purpose of doing so?

play eclipsify

This is a command that may be run from a terminal (command line) at the root of a Play application folder. The output from the command is a set of Eclipse configuration files and folders. This allows the app to then be imported into the Eclipse IDE. Among the files generated is .classpath. This establishes an absolute path (specific to the particular host machine) to the various Play library files (jars) required for the project. If the application is opened in a different computer then re-running play eclipsify will adjust the paths appropriately.

[9] Play uses a templating engine. Explain. Discuss where templating has been used in your project application, for example.

The Play template engine facilitates dynamic generation of "HTML, JSON or any text-based formatted document". It uses the Groovy language. Templates are stored in the app/views folder. For example, if we have a controller named Donation.java that has two methods, index and donate, then this will be matched to an app/views/Donation folder in which there will be two template (html) files named index.html and donate.html.

Play Documentation: Template Engine

[10] What is the role of the controller?

In the Play framework, a controller is a Java class derived from the Play Controller class and located in the controllers package. It provides a layer between the UI and the model and decides what to do with HTTP requests, generally applying changes to the model.

Play Documentation: Controllers.

[11] What is the role of the view?

The view is what the user is presented with. It facilitates interaction with the model (via the controller).

[12] What is the role of the model?

The model is the domain-specific representation of the data related to the application. It comprises one or more Java classes, derived from the (Play) Model class.

[13] Explain @Entity and the parent class Model as used in this code snippet:

@Entity
public class Donation extends Model 
{
  ...
}

Model is a Play support class that helps "to deal with JPA", the Java Persistence API. JPA is a specification for managing data in the presence of Java objects and a relational database. The implementation of JPA we use with Play is Hibernate.

Using @Entity as above results in the Donation class being mapped to a table (whose default name is Donation). The fields in Donation (not shown) become the columns in the database.

[14] Explain the following terms:

@ManyToOne
@OneToMany
@OneToOne

@ManyToOne annotation defines a many-to-one mapping. An example of this could be found in an Employee class and a Department class. A Department may have many Employees but any one Employee will belong to only one Department. This might be repesented as follows:

@Entity
public class Employee {
  @ManyToOne
  private Department department;
  // ...
}

The @OneToMany is the converse of @ManyToOne.

@Entity
public class Department {
  @OneToMany(mappedBy="department")
  private List<Employee> employees;
  // ...
}

An example of OneToOne mapping is demonstrated in this snippet:

@Entity
public class Employee {
  @OneToOne
  ParkingSpace parkingSpace;
  // ...
}
@Entity
public class ParkingSpace {
  @OneToOne(mappedBy="parkingSpace")
  Employee employee;
  // ...
}

Reference: Pro JPA 2, Mastering the Java™ Persistence API. By Mike Keith , Merrick Schincariol (Pages 92, 121, 124).

JavaScript Language (JS)

[1] Who developed the JavaScript language and when was it developed?

Developed over 10 days in May 1995 by Brendan Eich while working at Netscape (now Mozilla). Originally called Mocha by Marc Andreesen, the founder of Netcape.

[2] What have Java and JavaScript in common (other than the word Java)?

These are two completely different languages. Despite the implication of thescript suffix, JavaScript is a complete programming language. Both languages share the C-family syntax.

[3] What is ECMAScript? How does it compare to JavaScript?

ECMAScript is a language specification controlled by the non-profit standards organization Ecma International, formerly known as the European Computer Manufacturers Association (ECMA). JavaScript is an implementation of the ECMAScript standard. Other implementations are JScript, developed by Microsoft and ActionScript developed by Macromedia.

[4] What role does JavaScript play in developing client-side code?

Provides access to main components of web page such as CSS properties, markup content (div, img, p and so on). It facilitates communication to the server using forms.

[5] What is ES6? What preceded it? Name some new features introduced in ES6.

ECMAScript 2015 (6th Edition) is the current version of the ECMAScript Language Specification standard and is commonly referred to as ES6.

Some of the many new features include block scope, arrow functions, constants, spread operator, rest parameters, sets, maps and so on.

[6] JS now has 6 primitive types. What is a primitive type? List some of the six.

All types except objects define immutable values, that is values which are incapable of being changed. Values of these types are referred to as primitives. Here are the six: boolean, number, string symbol (introduced in ES6), null and undefined.

MDN Primitive values

[7] JS is a loosely typed language. Explain.

Loose typing means a variable is declared without specifying its type. It does not mean that the variable does not have a type. For example let s = "This is a string"; defines a string variable and let n = 10; declares a number variable. In neither case is the type explicity stated, this being determined at run time. This approach contrasts to strong typing where the compiler checks as much as possible before the program gets deployed.

A consequence of loose typing is that any variable may receive any kind of value. This then means that any function parameter can receive any kind of value and return any kind of value.

[8] Discuss any behavioural differences between the types var, const and let.

Both const and let are block-scoped whereas var is function scoped. Of the three, only var may be hoisted.

"The const declaration creates a read-only reference to a value. It does not mean the value it holds is immutable, just that the variable identifier cannot be reassigned." For example:

const intArray = [1, 2, 3];
intArray[0] = 4; // legal
console.log(intArray[0]);

intArray = [4, 5, 6]; // illegal: attempt to reassign intArray

MDN const

[9] Java has several number types, for example, int, long, float and so on. How many number types does JS have?

Just one, Number.

[10] Discuss the strengths and weaknessess of the JS number types.

Having just one number provides significant advantage over, for example, Java with its many number types where choosing a number type that is appropriate when the program is written may prove inadequate much later and generate error. For example, int type may be okay for years but prove too restrictive as data structures incrcease in size and cause overflow errors.

Unfortunately the JavaScript Number type is a float type and therefore unsuitable for industries such as banking and finance where 2 + 2 is expected to equal 4, a result not achievable with floating point types.

[11] What is the difference between null and undefined?

undefined is the default value for variables. If you declare a variable but do not give it an initialization value then it will be undefined. Also, all functions return a value. If a value is not specified then undefined is the returned value.

null "represents the intentional absence of any object value." MDN null.

Difference between null and undefined
-------------------------------------
typeof null        // object (bug in ECMAScript, should be null)
typeof undefined   // undefined
null === undefined // false
null  == undefined // true

[12] ES6 introduced the Symbol type. What is it and what is its purpose?

"A symbol is a unique and immutable data type and may be used as an identifier for object properties. The Symbol object is an implicit object wrapper for the symbol primitive data type." MDN Symbol.

[13] What is a JS object?

"An object is a collection of properties, and a property is an association between a name (or key) and a value".

MDN: Working with object

[14] What is an object literal?

A JavaScript object literal is a comma-separated list of name-value pairs wrapped in curly braces. It is the recommended method to create an object.

Example:

let anObject = {
  firstName: 'Homer',
  lastName: 'Simpson',
};

An array literal also exists:

const fruits = ["Apple", "Banana"];

console.log(fruits[1]); // Banana

[15] How do JS objects and arrays differ? When would one use an object, when an array?

Objects comprise name-value pairs, the name being the key.

An array is a single object that contains multiple values stored as a list. Each array element is accessed using an index number. For example, fruits[1] in an earlier question.

Thus, you use arrays when you have numerically indexed data.

[16] Discuss the use of semi-colons in JS.

The use of semi-colons is optional but recommended and in the case of some style guides such as AirBnB, obligatory. Some surprising errors may result if semi-colons are omitted.

[17] Discuss the differences, if any, between the uses of the equality operators in the following code:

String s1 = 'Hello world';
String s2 = 'Hello world';
let b1 = s1 == s2;
let b2 = s1 === s2;

=== is known as strict comparison operator.

== known as type-converting comparison operator.

When the strict comparison operator (===) is used, the result is true only if the operands are of the same type and the contents match.

When the type-converting operator (==) is used the operands are converted if they are not of the same type and then strict comparison is applied. Examples:

1 == 1; // true
'1' == 1; // true - type conversion

1 === 1; // true
'1' === 1; // false - operands not of same type

MDN Comparison operators

[18] Explain the terms truthy and falsy.

All expressions are either truthy or falsy.

An expression is falsy if it is any of the following:

false
undefined
null
0
NaN
"" (the empty string)

[19] Explain how the use of the && and || operators differ in JS compared to in Java.

Consider two variables a and b and a variable c, the result of applying the above operators. Then in Java:

c = a && b is true if both a and b are true
c = a || b is true if either a or b is true

In JavaScript the behaviour is quite different:

a && b is a if a is falsy otherwise it is b
a || b is a if a is truthy otherwise it is b

&& is known as guard operator. Here is an example:

// Use && as guard operator
const flight = {};
const airline = "BA";
const nationality = "UK";
// Use of guard operator prevents TypeError exception
console.log(flight.airline && flight.airline.nationality); // > undefined
console.log(flight.airline.nationality); // TypeError exception (no guard operator)

|| may be used to insert default values. Here is an example:

// Use OR to insert default values
const person = {};
let name = person.name || 'No such person';
console.log(name); // > No such person

person.name = 'Jane Doe';
name = person.name || 'No such person';
console.log(name); // > Jane Doe

Very similar to in Java using logical operators, if, while, do-while, for, forEach, break and continue.

Externally located script code is the preferred option.

A function is a block of code defined once and invokable many times. Such a block of code located witin an object is referred to as a method.

Functions have four parts: the reserved word function, a name (optional), zero or more parameters and a block of statements with an optional return statement, contained within curly braces.

[25] Discuss some of the implications in using strict mode.

Introduced in ES5, strict mode is a means of opting in to a restricted variant of JS. For strict mode to operate as designed a browser needs to support it. Some of the changes strict mode causes are:

  • eliminates some JS silent errors,
  • makes it impossible to accidentally create global variables by, for example, omitting var, let or const keywords when first declaring a variable.

Examples:

// Assignment to a getter-only property
var obj2 = { get x() { return 17; } };
obj2.x = 5;  
console.log(obj2.x); // > 17
'use strict';
// Assignment to a getter-only property
var obj2 = { get x() { return 17; } };
obj2.x = 5; // throws a TypeError
console.log(obj2.x); // > Uncaught TypeError: Cannot set property x of #<Object> which has only a getter

MDN: Strict mode

[26] What is the arrow function? How does the behaviour of the this object or reference change when using an arrow function?

An arrow function is a more concise syntax compared to function expressions. For example:

function x() {
  function inner() {
    let x = this;
  }
}

becomes:

() => {
  function inner() {
    let x = this;
  }
}

An arrow function is always anonymous.

Importantly, this binding is different. 

'use strict';

let myObj = {
  x: 0,
  set: function (x) {
    let _this = this; // workaround ...
    modify(x);
    function modify(val) {
      _this.x = x; // ... to access myObj this
    };
  },
};
myObj.set(100); // sets myObj.x to 100
//this now bound to containing object myObj
'use strict';
let myObj = {
  x: 0,
  set: function (x) {
    let modify = (val) => { // nested function
      this.x = val; // this now bound to myObj
      console.log(this); // Object{x: 0}
    };

    modify(x);
  },
};
console.log(myObj); // Object { x: 100, set: myObj.set() }
myObj.set(100); // myObj.x set to 100
console.log(myObj.x); // 100

[31] In the context of a JS file, what do the terms minification and obfuscation mean and why might these processes be used?

Obfuscation is the process of modifying source code without altering its functionality so that it becomes difficult to read. Some reasons for obfuscation of code are secrecy and prevention of tampering.

[33] JS is referred to as a prototypal language. Explain. Compare to Java.

Java is a classical language: Classes are a fundamental feature. Classes inherit from classes. JavaScript does not have classes. It is an object oriented language that inherits directly from other JS objects. It is true that ES6 introduces the concept of classes but this is merely what is referred to as syntactic sugar.

[34] What does the term scope mean? How do scopes differ in the JS and Java languages?

Scope is the current context of execution in which values and expressions may be validly referenced. In simple terms, a variable is in scope at a particular point in the execution cycle if it is visible, that is accessible.

Java has block scope. JavaScript has both global, block and function scope. Block scope was introduced in ES6.

[35] Explain and discuss in the context of JS the terms global scope, function scope and block scope.

A variable that has global scope is visible throughout the entire program.

A variable that has function scope is visible throughtout the function within which it is declared irrespective of where in the function its declaration takes place. 

// Example function scope
'use strict';

let myObj = {
  set: function (val) {
    modify(val);
    function modify(v) {
      x = v; // legal because x has function scope
      console.log(x);
    }

    var x;
  },
};
myObj.set(100);

A variable has block scope if it is visible only inside the pair of curly braces within which it is declared.


// Example block scope
'use strict';

function x() {
  {
    let b = 20; // b is visible only within its enclosing { }
  }

  console.log(b); // Uncaught ReferenceError: b is not defined
}

x();

[36] What is an immediately invoked function expression (IIFE)? When might one use an IIFE?

Sometimes referred to as a self-executing anonymous function. Here are examples of an IIFE and a standard JS function.

'use strict';

// IIFE
(function () {
  let c = 20;
  console.log(c); // > 20
}());

// JS function expression
function x() {
  let b = 10;
  console.log(b); // > 10
}

x(); // function invocation

[37] Describe some techiques to avoid polluting the global scope (namespace)?

  • Use strict mode.
  • Use an IIFE.
  • export and import statements (not yet natively implemented in browers).

[38] Explain Temporal Dead Zone (TDZ).

In ECMAScript 2015, let will hoist the variable to the top of the block. However, referencing the variable in the block before the variable declaration results in a ReferenceError. The variable is in a "temporal dead zone" from the start of the block until the declaration is processed.

function do_something() {
  console.log(foo); // ReferenceError
  let foo = 2;
}

MDN: Temporal dead zone and errors with let

[39] Explain what is meant by a closure. Describe an example where use of a closure would be advantageous.

"Closures are functions that refer to independent (free) variables (variables that are used locally, but defined in an enclosing scope). In other words, these functions 'remember' the environment in which they were created."

In the example below, the functions add5 and add10 are closures.


function makeAdder(x) {
  return function(y) {
    return x + y;
  };
}

var add5 = makeAdder(5);
var add10 = makeAdder(10);

console.log(add5(2));  // 7
console.log(add10(2)); // 12

Closures are particularly useful in web programming where much of the code is event driven. We have seen several examples in the JS slide decks and encountered applications of closures in the Google maps code, for example where we click on markers to reveal info windows.

MDN: Closures

Git Versioning System

[1] Git is a distributed versioning system (DVCS). How does this compare to centralized version control systems (CVCS)?

Declaration of interest: I am biased in favour of DVCS, specifically Git.

In a centralized version control system there exists a central single copy of the project, usually on a server accessible by the development team. Some of the disadvantages are:

  • single point of failure: if the server goes down the development team may be seriously disrupted.
  • remote commits are slow.
  • unsolicited changes may break your build.
  • merging is not simple.

Some common centralized version control systems are CVS, Subversion (or SVN) and Perforce.

In a DVCS each developer has a full copy of the repository that includes the full hisory of the project. Generally, but not necessarily, there will be a central copy of the repository. This might be hosted on a private server or on proprietary server, for example as provided by Bitbucket or Github.

  • the single point of failure weakness of CVCS is avoided.
  • the number of remote commits to the central copy can be minimised.
  • the developer controls when to accept other developers' code changes.
  • merging is simple.

A disadvantage is that DVCS is a complex application with a steep learning curve.

The most common DVCS are Git, Mercurial and Bazaar.

[2] Describe in broad terms how to put an existing application under Git version control.

  • Change into the project folder in a terminal.
  • Create a .gitignore file to control what content to exclude from the repository (repo).
  • Initialize an empty repo.
  • Add the files to the repo's index.
  • Commit the files.
  • Push to a remote copy (optional).

[3] Would you put all files in a project under Git control? If not, why not? How would you decide which files to exclude and how would you achieve this?

Use a .gitignore to specify which files within the project should be excluded from the repo. Only those files that are essential to build the project are committed. All temporary files, binary files, user-specific project and configuration files and so on are excluded.

[4] What do you hope would happen were you to run this command?

git push origin master

Pushes a current snapshot of the master branch to the remote repository.

[5] What do you hope would happen were you to run this command?

git pull origin master

Refresh the local working directory (tree) by fetching the latest snapshot from the remote repository and fast forward merging it with the local working tree.

[6] What is the index?

The staging area or cache in which files are placed before committing to the repository. For example, git add . adds all new and modified material to the index.

[7] What does the commit command achieve?

Creates and saves a snapshot of the latest state of the working tree in the repository.

[8] What are the consequences for your working folder and its contents when, while on the master branch, you run a command such as the following. What might dev refer to?

git checkout dev

dev could be either a tag name or a branch name. It either event, running the command will checkout the repository at that tag or branch. If dev is a tag then the result would be a detached head.

[9] How many object types in Git, list them and briefly explain?

Git has four objects, Tree, Blob, Commit and Tag.

  • Tree: equivalent to file or folder
  • Blob: stores data. Corresponds to file contents. May include folders and subfolders.
  • Commit: describes the commit
  • Tag (light & annotated): describes the tag. A lightweight tag is a pointer to a specific commit. An annotated tag is stored as a full object and contains information such as the tagger name, email, tag date and so on.

[10] What is SHA-1, a SHA-1 hash and what role do they play in Git?

SHA-1 (Secure Hash Algorithm 1) is a cryptographic hash function designed by the United States National Security Agency (NSA). It produces a 160-bit (20-byte, 40-character) hash value known as a message digest. In Git this digest is used as a key in the Git object key-value pair.

[11]. The merge and rebase commands may be used, for example, to move a development branch to the master branch. Which command should one use and why?

Merging is non-destructive and so is a safe operation. However, other developers may find it difficult to understand the history of the project.

Rebasing provides for a clean, linear history but should not be applied to a public branch. That is, if anybody else is looking at the branch being considered for rebasing then don't rebase.

[12] How might one revise the commit history of a project, for example in reducing the number of commit messages or changing the message content of a message or messages? What is good practice in doing so?

Use the interactive rebase command and squash the surplus commits. Example:


rebase -i HEAD~3

This will facilitate compressing the last three commits into one.
