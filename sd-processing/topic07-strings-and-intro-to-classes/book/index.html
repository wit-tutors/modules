Exercise 1

Write code (using the equals() and substring() methods) that sets a string called name. The name should be read using a JOptionPane. The code should print to the console ''First name is John'' if the first four characters of the name is John and ''First name is not John'' otherwise.

Exercise 2

Looking at the Spot constructor 

Spot(float xPos, float yPos, float diamtr){
    xCoord = xPos;
    yCoord = yPos;
    diameter = diamtr;
  }

Rewrite the constructor so that the constructor imposes validation rules in the value of xPos and yPos, i.e. that both xPos and yPos must be non-negative values. So if the value of the xPos and/or yPos is negative, you should give it a value of 0 instead of the input value.

Exercise 3

If we add another field to Spot - a String field called spotName. (This simple stores the name of the spot). The class now looks like (without the validations done above)

class Spot{
  String spotName;
  float xCoord, yCoord;
  float diameter;

 Spot(String name, float xPos, float yPos, float diamtr){
    spotName = name;
    xCoord = xPos;
    yCoord = yPos;
    diameter = diamtr;
  }
}

and the ''calling program'' looks like:

Spot sp;

void setup()
{
  size (100,100);
  noStroke();
  sp = new Spot( "SpottySpot", 33, 50, 30);
}

void draw()
{
  background(0);
  ellipse(sp.xCoord, sp.yCoord, sp.diameter, sp.diameter);
  println(sp.spotName);
}
  • Firstly, run this program. See how the code in the class and calling program have changed to adapt to the extra field.
  • Then, add a validation to the constructor that only allows names that are no more than 7 characters long. If the name is too long, the constructor should impose the rule by cutting the spotName string to the first 7 characters.
  • Test this on the calling program as given.
