Setters and Getters - validations

Using validation in constructors

  • Using the ''File'', ''Save as...'' menu options, save this sketchbook as ''Spot_Version7_1''.
  • Replace the code in your (3 parameter) Spot constructor with the following code(don't cut and paste it, type it in):

Spot(float xCoord, float yCoord, float diameter)
  {
    this.xCoord = xCoord;
    this.yCoord = yCoord;
    if ((diameter > 0) && (diameter < 500)){
       this.diameter = diameter;
    }
    else{
       this.diameter = 10;
    } 
  }

  • This code will only allow values of between (and including) 0 and 500 into the diameter field. If values outside this range are send in via parameter, then the value of 10 is assigned to the diameter field (using the else clause).

  • To see this validation in action, write the following code into the 'user' tab (called ''Spot_Version7_1'')

Spot sp;

void setup()
{
  size (100,100);
  noStroke();
  sp = new Spot(33, 50, 30000);
  println("The value of diameter is : "+ sp.getDiameter());
}

void draw()
{
  background(0);
  sp.colour(255,0,0);
  sp.display();
}

Use of validations in setters

  • We use setters as the only safe way to change the values of fields after the object has been created (we can use the constructor validation rules at creation stage).
  • We look at the setter for the value of the diameter field.
  • A reasonable validation rule would be that this filed should be in the range 1..500. (anything bigger would mean the spot woud be too big for the screen)
  • Without this validation rule, the setter method (in Spot) looked like this: 
  public void setDiameter(float diameter){
        this.diameter = diameter;
  }
  • Using the validation rules above, the code should read: 
  public void setDiameter(float diameter){
    if ((diameter > 0) && (diameter < 500)){
        this.diameter = diameter;
    }
  }

  • Note:

    • The use of the AND (&&) in the condition
    • There is no else in the if statement. This means that if the parameter is not in the correct range, then the diameter field will remain unchanged. This is the convention we use for this module.

  • To see how this works, write the following code in the setup method (we are simply changing the call of the setter to try to change the value of diameter to an invalid (40000) value).

void setup()
{
  size (100,100);
  noStroke();
  sp = new Spot(33, 50, 30);
  println("After constructor, the value of the diameter is " + sp.getDiameter());
  // Now change the value of the diameter
  sp.setDiameter(40000);
  println("After setter, the value of the diameter is " + sp.getDiameter());
}
