Specification

In this assignment, you are asked to write code to produce a working animation of your choice in Processing (Java).

The aim of the assignment is for you to demonstrate your understanding of structures seen so far, so you are asked to include the following :

  • At least one user-defined class, adhering to encapsulation rules, complete with:
    • fields
    • constructors
    • getters
    • setters
    • other methods
  • The usual Processing class containing the draw(), setup(), etc. This class must use your user-defined class(es) in an appropriate manner.
  • Use of String methods.
  • Use of primitive arrays.

Refer to the Pong game covered in lectures and labs for inspiration.

The complexity of the entire code is important, i.e. the more complex the code, the more marks you will achieve. (see marking scheme)

Marking Scheme

There are three components to your overall mark:

  • Structure of code
  • Complexity of code
  • Interview mark

Structure of Code (100 Marks)

  • User-defined class (at least one):
    • Creation of user-defined class with appropriate fields, constructors, getters, setters, other methods(25 marks).
    • The user-defined class(es) are used in an appropriate manner (15 marks).
  • Working animation i.e. inclusion of the usual Processing class containing the draw(), setup(), etc (20 marks):
  • Data structure:
    • Use of a data structure to store information i.e. a primitive array (15 marks).
    • Calculations performed on the information stored (10 marks).
  • Use of String methods (5 marks).
  • Comments, indentation, naming, structure of code, etc (10 Marks)

Complexity of code (10 Marks)

This is a mark out of 10 based on the complexity of the problem solved/code. So you will get more marks if the animation is more involved than one that is very simple. Please do not confuse an elegant solution of a complex problem with a simple problem.

Interview Mark (10 Marks)

You will be asked to explain your work during study week. This is to ensure that the work is your own. You may be asked (for example) to explain:

  • Any of the code.
  • How you decided on your animation.
  • How you developed your animation; where did you start first, etc?
  • Anything else.

How is the final mark calculated?

The final mark is calculated by multiplying the three component marks and dividing by 100. This means that each of the marks is used as a multiplier so a weakness in any of the components will negatively affect your grade.

Example:

  • Structure of code = 80/100
  • Complexity of code = 9/10
  • Interview Mark = 10/10
  • Final mark = (80 by 9 by 10)/100 = 72%

Note

Specifically, the multiplier effect means that if you do not present for your interview you will receive 0 for the interview and 0% overall. This means you must be avaiable for interview during study week.
